Some significant words in AngularJS

$rootScope - it's just a scope, that has every single app, all other scopes are descendant of it.

$inject - this is one of three ways of annotating code with some name info, like:

It's called $inject property annotation. We need to annotate functions so that the injector knows what services to inject into the function.

$q - well, it's an object in AngularJS that works with promises. In particular, $q.all is used for concurrent function's implementations. For a example:

$stateParams - this is an object (in ui-router) that have one key per url parameter. That obj only contain the params that were registered with that state. But we can use a resolve statement in the parent route to avoid that thing. I mean in 'resolve' we can return some certain stateParams and use it in other states, like this:

$cookies - this's just a service that provides read/write (get/put(!)) access (methods) to browser's cookies.

$parse - the docs are telling me that it converts AngularJS expressions into a function, but at this moment I'm still a bit confused about that explanation. Will c.

$http - well, here we go without any surprises, it's just a func which takes an object that is used to generate an HTTP req and returns a promise.
